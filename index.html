Frequently Asked Questions

1. the difference between var, let, and const -

var is function-scoped and can be redeclared and updated; it may cause unexpected bugs due to hoisting.
let and const are block-scoped; let can be updated but not redeclared, while const cannot be updated or redeclared.

2. the difference between map(), forEach(), and filter() +

map() returns a new array by applying a function to each element.
forEach() executes a function on each element but does not return a new array.
filter() returns a new array containing only elements that meet a condition.

3. explain arrow functions and how they are different from regular functions +

Arrow functions have a shorter syntax and do not have their own this, inheriting it from the surrounding scope.
Regular functions have their own this, can use arguments, and work as constructors, unlike arrow functions.

4. how JavaScript Promises work +

JavaScript Promises represent asynchronous operations and can be in one of three states: pending, fulfilled, or rejected.
They use .then() for success, .catch() for errors, and .finally() for cleanup after completion.

5. how closures work in JavaScript +

A closure is a function that remembers the variables from its outer scope even after the outer function has finished executing.
It allows data encapsulation and is useful for creating private variables in JavaScript.
